The class "RegisterItem"  specifies elements of information to be recorded for each item held in a register. The technical standard that specified an item class may specify additional elements to be recorded.<br/>

Notes: The attribute "itemIdentifier" is represented as a Character that is used to uniquely denote that item within an item class and is intended for information processing. The class/identifier union shall be unique within the register.<br/><b>NOTE</b> When a register contains items from different item classes, each item will be uniquely identifiable by the item identifier alone.<br/>

«interface» RegisterItem
Class   		Name: predecessor
The role "predecessor" describes a previous concept of the "RegisterItem" 		Name: successor
The role "sucessor" describes a later concept of the "RegisterItem"
Details:
The association "Modification" connects the RegisterItem to one or more other instances of RegisterItem that preceded or superseded it. The existence of more than one successor for a registered item implies a subdivision of the concept represented by that registered item. Any "<i>successor"</i> shall represent the same concept as its "predecessor" or a sub-concept of that concept.<br/><br/>EXAMPLE: The feature type “buoy” held in one feature catalogue register might be replaced by several feature types representing subtypes of “buoy” in another register. Conversely, several types of “road” in one register might be replaced by a single supertype “transportation route” in another feature register.<br/>

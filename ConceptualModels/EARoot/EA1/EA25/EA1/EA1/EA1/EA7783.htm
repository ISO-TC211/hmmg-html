This empty abstract class is the root of the address specification tree. The design strategy is to define "address elements" commonly used across multiple cultures and countries, and to allow any country to define an address type of its own by combinations of these elements. New elements are allowed if they have character string representations. The printed address for any country would be a template of the strings specified in  its subtype of address. Note that any viable instance for any subtype of abstract address would also be a viable AD_Address, as long as all of its address elements are defined. <br/>

<br/>The LC_ValueObject is a metalanguage object that describes how permitted numeric values at the metalanguage level maybe instantiated to the basic number types form ISO/TS 19103 at the type level. The LC_ValueClass metalanguage object is a supertype of the three value metalanguage value objects LC_PermittedRealValueType, LC_PermittedPercentageValueType and LC_PermittedIntegerValueType. Each of these value objects has subtypes that describe a number and a range. Each of the value objects is described by a constraint that characterises the type of numerical entity that may be represented by the value object. <br/><br/>These value objects are generators for the actual value objects that will appear in instances of a classification system generated using the LCML. The result of instantiating these objects are value classes taken from the basic types provided in ISO/TS 19103. The results of instantiating a classification system are objects that carry actual values. In the various examples given in this international standard value objects have been instantiated from the metalanguage level to the type level and then to the instance level, to produce real objects as needed in an example.<br/><br/>It has three subtypes: LC_PermittedRealValueType, LC_PermittedPercentageValueType, and LC_PermittedIntegerValueType.<br/><br/>

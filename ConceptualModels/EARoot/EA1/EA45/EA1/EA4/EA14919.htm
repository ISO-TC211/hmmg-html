AddressComponent
Class   		Name: valueComponent
Identifies the subordinate address component. NOTE ‘within scope of’ refers to the relationship between a superordinate address component and a subordinate address component in an address reference system. EXAMPLE 1 Thoroughfare names could be within scope of a place name. If thoroughfare names are unique within a place name, the thoroughfare name component is a valueComponent of the place name component. EXAMPLE 2 Building names could be within scope of a thoroughfare name. If building names are unique along a thoroughfare name, the building name component is a valueComponent of the thoroughfare name component. EXAMPLE 3 Box numbers could be within scope of a post office. If each box number at a post office is unique, the box number component is a valueComponent of the post office component. 		Name: scopeComponent
Identifies the superordinate address component. NOTE ‘within scope of’ refers to the relationship between a superordinate address component and a subordinate address component in an address reference system. EXAMPLE 1 Thoroughfare names could be within scope of a place name. If thoroughfare names are unique within a place name, the place name component is the scopeComponent of the thoroughfare name component. EXAMPLE 2 Building names could be within scope of a thoroughfare name. If building names along a thoroughfare are unique, the thoroughfare name component is the scopeComponent of the building name component. EXAMPLE 3 Box numbers could be within scope of a post office. If each box number at a post office is unique, the post office component is the scopeComponent of the box number component.
Details:
The reflexive withinScopeOf association connects an AddressComponent with a superordinate AddressComponent (the scope component) or subordinate AddressComponents (the value components).<br/>
Address
Class   		Name: addressComponent
A component that is a constituent part of the address. 		Name: address
An address of which the address component is a constituent part.
Details:
The comprises association connects an Address with its constituent parts (AddressComponents). In the other direction, the comprises association connects an AddressComponent with all the Addresses of which it forms a constituent part.<br/>

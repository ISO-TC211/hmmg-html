The following optional attribute was omitted from the specialized class:<br/>volume. Consideration of a 2D type for BoundaryFaceString objects (geometrically represented by GM_MultiCurve) comes from the conceptual definition of a BoundaryFaceString as a set of connected vertical faces defined through a linear object.<br/>If values for Area should be derived from the BoundaryFaceString (namely through the computeArea() operation), then this attribute should be omitted. No special constraints are defined regarding the operations,  hence they are not shown, but are inherited "as is" from the superclass.<br/><br/>

The LC_Element object is a generalization of a large number of subtypes. These subtypes form the basic elements of the LCML metalanguage model. A classification system, described in terms of the meta language, consists of land cover classes formed as searalizations of the LCML metalanguage model LC_Element subtypes in various combinations. <br/><br/>If the LC_Element object is used with no subtypes it describes an area with no data.<br/><br/>This object has two relationships:<br/><br/>- It is an element in the ordered relationship stratumComposition with the LC_Stratum metaobject.<br/><br/>- There is an <i>inSameStrataRelationship</i> between separate instances of the LC_LandCoverElement metaobjects.  The elements in the same strata are regulated by four different relationship statuses: Fixed, Exclusive, Conditional and Sequential Temporal as described in LC_ElementPresenceType which is an enumerated values class for the attribute <i>presenceType</i> in the association class LC_ElementRelationInSameStratum.<br/><br/>The constraint on the relationship <i>stratumComposition</i> is that {the sum of the cover of all the LC_Elements within one stratum can not be more than 100%}. <br/><br/>The constraint on the <i>inSameStrataRelationship</i> is that {an LC_Element may have a sequentialTemporalRelationship only with the next one in the ordered list of LC_LandCoverElements which belong to the same LC_Stratum}<br/>

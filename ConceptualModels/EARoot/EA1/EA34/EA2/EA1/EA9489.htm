The association class LC_ElementRelationInSameStratum defines the relationship between instances of the LC_Element object. <br/><br/>The association class LC_InterStrataRelationship has three attributes <i>presenceType</i>, <i>temporalType</i> and <i>lengthOfTemporalRelationship</i>. The permitted values of the attribute <i>presenceType</i> are described by LC_ElementPresenceType. The permitted values of the attribute <i>temporalType</i> are described by the value object LC_SequentialTemporalRelationshipType. The permitted values of the attribute <i>lengthOfTemporalRelationship</i> are defined by the values object LC_PermittedPosRealRange. <br/><br/>The attribute <i>presenceType</i> allows one to indicate whether an LC_Element in a stratum is fixed (mandatory); that is, it is always present in the composition of elements for an LC_Stratum and indirectly for a LC_LandCover object, or whether it is exclusive, conditional (optional) or whether a sequential temporal relationship exists.  This allows one to model a situation where a certain LC_Element is permitted but not always present in the definition of an LC_Stratum and indirectly an LC_LandCoverDescriptor object; that is it is optional, and also the situation where one of two LC_Elements is permitted but not both (exclusive OR - XOR). <br/><br/>Optionality useful when describing a LCCS that contains classes or legends based on ambiguous definitions or when one wants to indicate that a further refinement of the land cover class is possible.<br/><br/>The <i>sequentialTemporalRelationship </i>attribute value indicates a temporal relationship between LC_Elements. One element may exist at one time and another element at a different time. For example, in a nursery of pinus trees, when the plants are very young they might be classified as shrubs because they are less than 2 meters in height, however the description of the land cover can indicate that after som time (some years) the element will become trees.<br/><br/><br/><br/>
